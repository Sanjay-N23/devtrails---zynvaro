README and presentation documents must be visually stunning, creative, and jaw-dropping — not generic.

**Why:** International hackathon with 10,000+ participants. The README is the first thing judges see. Creative presentation + clarity = winning combination. Danish explicitly rejected a "normal" README as insufficient.

**How to apply:** Always use advanced GitHub features: Mermaid diagrams, shields.io badges, capsule-render headers/footers, LaTeX math, collapsible sections, alerts/admonitions, creative tables with emojis, sequence diagrams, flowcharts, Gantt charts, pie charts. Prioritize both creativity AND clarity — judges must understand the solution quickly. Map content to evaluation rubrics indirectly.
